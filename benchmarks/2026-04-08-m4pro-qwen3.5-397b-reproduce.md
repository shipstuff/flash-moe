# FlashMoE Benchmark Results — 2026-04-08 (fresh runs, reproducible)

**Hardware:** Apple M4 Pro (Mac Mini)
**Model:** Qwen3.5-397B-A17B-4bit (K=4 experts/layer)
**Binary:** Clean committed HEAD (`614ae10`), freshly built `make clean && make -j4`
**Method:** Each config run 3 times, reported range + median. Tokens generated only (TTFT excluded).
**OS page cache:** Warm — run once before measurement window.

## Flags Reference

| Flag | Effect |
|------|--------|
| `--malloc-cache N` | Pre-allocate N malloc-backed Metal buffers for expert weights (zero-copy, bypasses OS page cache) |
| `--2bit` | Use 2-bit quantized experts (packed_experts_2bit/) instead of 4-bit |
| `TQ_KV=0` env | Disable TurboQuant KV cache (float K/V) |
| `TQ_KV=1` env | Enable TurboQuant KV cache (default) |

## Observations

1. **malloc-cache-64 dominates**: ~88-172% improvement across all generation lengths. Pre-allocating 64 expert weight buffers eliminates all I/O bottleneck during generation.

2. **TQ is neutral for generation**: TurboQuant KV helps at prefill (TTFT) but doesn't help generation throughput on M4 Pro. Expert I/O (1.17ms/layer) dominates the critical path, not KV bandwidth.

3. **2-bit is a step below malloc-cache**: 14.4-14.5 tok/s vs 15.6 tok/s. Still 74% faster than baseline but not as good as 4-bit + malloc-cache.

4. **512tok sees slight drop** (14.75 vs 15.61 at 48tok) — likely KV cache pressure as sequence grows, but still 2.7× baseline.

5. **No cold-start penalty with malloc-cache**: The first run with malloc-cache-64 already hits 15.56-15.72 tok/s, vs baseline first run at 6.02 (cold page cache). The malloc buffers are pre-warmed at startup.

---

## TurboQuant KV Status (M4 Pro, generation workload)

- TQ kernel latency: p99.99 = 0.424ms, p50 = 0.320ms, zero stall candidates
- TQ is stall-free on M4 Pro's unified memory architecture
- But TQ does not improve generation tok/s — the bottleneck is expert I/O (1.17ms/layer), not KV bandwidth
- TQ helps TTFT / prefill significantly (earlier results showed 53% TTFT improvement with batch prefill)
- **Recommendation**: Use TQ for long prompts where TTFT matters; for pure generation speed, `--malloc-cache 64` alone is best
