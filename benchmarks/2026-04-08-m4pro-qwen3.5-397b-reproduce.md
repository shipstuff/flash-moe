# FlashMoE Benchmark Results — 2026-04-08 (fresh runs, reproducible)

**Date:** 2026-04-08
**Hardware:** Apple M4 Pro (Mac Mini)
**Model:** Qwen3.5-397B-A17B-4bit (K=4 experts/layer, 60 layers)
**Binary:** `9098bd9` (benchmarks commit), freshly built `make clean && make -j4`
**Method:** Each config run 3 times, reported range + median. Tokens generated only (TTFT excluded).
**OS page cache:** Warm — run once before measurement window.

## Flags Reference

| Flag | Effect |
|------|--------|
| `--malloc-cache N` | Pre-allocate N malloc-backed Metal buffers for expert weights (zero-copy, bypasses OS page cache) |
| `--2bit` | Use 2-bit quantized experts (packed_experts_2bit/) instead of 4-bit |
| `TQ_KV=0` env | Disable TurboQuant KV cache (float K/V) |
| `TQ_KV=1` env | Enable TurboQuant KV cache (default) |

## Key Findings

1. **malloc-cache-64: ~170% faster than baseline at 512tok** (5.48→14.82 tok/s)
2. **TQ is neutral for generation**: TQ_KV=0 and TQ_KV=1 give identical throughput. TQ was benchmarked as part of the baseline but does not affect generation speed — it helps TTFT/prefill only.
3. **2-bit is good but not as good as malloc-cache-64**: 13.76 tok/s vs 14.82 tok/s at 512tok
4. **No cold-start penalty with malloc-cache**: buffers are pre-warmed at startup
